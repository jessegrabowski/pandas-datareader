.. _whatsnew_0110:

v0.11.0
-------

Highlights include:

New Features
~~~~~~

- Every reader and ``DataReader`` accept ``output_type`` selecting the dataframe library of the
  returned data: ``'pandas'`` (default, unchanged output), ``'polars'``, ``'pyarrow'`` (alias
  ``'arrow'``), or ``'dask'``. Non-pandas backends receive tidy frames -- dates and identifiers as
  plain columns, multi-symbol data one row per (date, symbol) -- and install via extras, e.g.
  ``pip install pandas-datareader[polars]``. See :ref:`remote_data.output_backends`.
- Multi-symbol daily reads fetch concurrently; the new ``max_workers`` argument (default 5)
  controls the thread count. Pass ``max_workers=1`` for sequential fetching or when supplying a
  session that is not thread-safe. See :ref:`remote_data.concurrent`.
- ``read_jsdmx``, ``read_jstat``, and ``MoexReader.read_all_boards`` accept ``output_type``.

Backwards incompatible changes
~~~~~~~~~~~~~~~

- ``narwhals>=2.0`` is a new required dependency.
- Alpha Vantage time-series output is indexed by a real ``DatetimeIndex`` instead of date strings,
  and the date-range filter compares datetimes rather than slicing strings lexicographically.
- Third-party subclasses of the reader base classes that override ``read()`` should override
  ``_read_core()`` instead; ``read()`` now dispatches payloads to per-backend presenters.

Other changes
~~~~~~
- The ``chunksize`` argument of the daily readers is unused and retained only for backward
  compatibility.
